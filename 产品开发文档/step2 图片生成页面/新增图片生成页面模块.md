````markdown
# AI Prompt Structurer
## PRD v2.4 — Image Generation Module

---

## 1. Product Overview

AI Prompt Structurer 是一个 Web 应用，用于：

1. 将自然语言解析为结构化字段
2. 根据模板生成标准化 Prompt
3. 使用 Prompt 调用图像模型生成图片

系统核心流程：

```text
Text → Prompt → Image
````

Prompt 是两个模块之间的数据连接层。

---

## 2. System Modules

系统包含两个核心模块：

```text
Text Extraction
Image Generation
```

模块关系：

```text
Text Extraction
        ↓
     Prompt
        ↓
Image Generation
```

职责划分：

**Text Extraction**

```text
Text → Prompt
```

**Image Generation**

```text
Prompt → Image
```

## 5. Text Extraction Module (Existing System)

路径：

```text
/extract
```

该模块已经实现以下流程：

```text
User Input Text
        ↓
LLM Extraction
        ↓
JSON Repair
        ↓
Zod Schema Validation
        ↓
Template Rendering
        ↓
Prompt Generation
```

结果保存到数据库：

```text
extraction_results
```

核心字段：

```text

json_data
prompt
created_at
```

其中：

```text
prompt
```

为最终生成的提示词。

## 13. API Design

### 13.1 获取 Prompt

```http
GET /api/extraction-results
```

返回：

```json
[
  {
    "id": "...",
    "prompt": "...",
    "created_at": "..."
  }
]
```

### 13.2 图片生成

```http
POST /api/generate-image
```

请求：

```json
{
  "extraction_result_id": "...",
  "model_id": "...",
  "image_size": "1024x1024"
}
```

服务器执行：

```text
load extraction_results.prompt
↓
call image model
↓
download image
↓
upload storage
↓
save result
```

返回：

```json
{
  "image_url": "..."
}
```

## 26. Key Notes for Codex

以下约束需要在开发时重点遵守：

1. `/extract` 与 `/generate-image` 是并列页面。
2. Text Extraction 页面只需要在 PRD 中简单带过，不作为本次主要改造对象。
3. 本次主要实现的是 Image Generation 模块。
4. Image Generation 模块调用的是 Text Extraction 已经生成并保存的 `extraction_results.prompt`。
5. 如非必要，不要修改原有文本提取页面的设计、布局、交互和处理链路。
6. 不要把图片生成功能塞进 `/extract` 页面内部。
7. 不要为了图片生成功能重新设计 Prompt 生成机制。
8. 如果需要新增跳转入口，最多只允许在 `/extract` 结果区域增加轻量入口，例如“前往图片生成”，但不是必须项。
